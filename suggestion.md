这是一个非常关键的洞察！**“编辑即导出参数配置”** 这个认知彻底改变了产品的设计逻辑。这意味着这个工具本质上不是一个非线性编辑软件（像 Premiere），而是一个 **“录屏 + 转码/封装工具”**。

基于**加入列表页**和**合并编辑与导出**这两个核心点，我为你构思了一套全新的**“二段式”产品架构**建议：

---

### 核心理念：从“流水线表单”转变为“资源管理器 + 沉浸式工坊”

我们将应用拆分为两个核心界面：

1. **Dashboard（项目列表页）：** 管理历史记录、文件状态。
2. **Studio（工作台页）：** 沉浸式的录制与后期保存环境。

---

### 1. 新增页面：Dashboard（项目列表页）

这是用户打开软件看到的第一屏，解决“我是谁？我在哪？我以前录的东西呢？”的问题。

* **布局建议：**
* **顶部：** 巨大的 **“新建录制” (New Recording)** 按钮（主操作）。
* **主体：** 一个表格或卡片流，展示历史项目。
* **列表字段：**
* 缩略图（视频第一帧）。
* 项目名称/ID（支持重命名）。
* 录制时长。
* 状态标签（已导出、草稿、转码失败）。
* 操作栏（下载、删除、**重新编辑/导出**）。




* **交互逻辑：** 点击“新建”进入 Studio 页面的录制状态；点击“历史项目”进入 Studio 页面的导出预览状态。

---

### 2. 改造页面：Studio（工作台页）—— 模态化设计

现在的页面是从上往下滚动的（录制->编辑->导出），建议改为**同一个页面，两种状态（State）**。

#### **状态 A：录制态 (Recording Mode)**

当用户点击“新建录制”时进入此状态。

* **视觉焦点：** 屏幕中央是巨大的**预览窗口**（显示当前摄像头或选定屏幕）。
* **操作栏（底部悬浮）：**
* 麦克风/系统声音开关。
* 录制区域选择（全屏/窗口）。
* **红色圆形的 [开始录制] 按钮**（核心）。


* **隐藏干扰：** 此时**完全隐藏**“编辑”和“导出”参数。用户在录制时不需要关心码率是多少，projectId 是多少，专注于“录”这件事。

#### **状态 B：预览与导出态 (Review & Export Mode)**

当用户点击“停止录制”，或者从列表页点击历史项目时，进入此状态。**这里将“编辑”和“导出”合二为一。**

* **布局结构（左右分栏）：**
* **左侧（2/3 区域）—— 播放器：**
* 显示刚才录好的视频。
* 提供简单的时间轴（Timeline），用户可以拖动首尾滑块进行裁剪（Trimming）。**这就是你说的“编辑”部分，它直接作用于视频预览。**


* **右侧（1/3 区域）—— 导出面板 (Export Panel)：**
* 这里放置原先“编辑”和“导出”里的所有参数。
* **项目名称**（原 Project ID，改为可读的标题）。
* **导出设置：** 分辨率、帧率、码率（提供“高/中/低”预设，折叠高级参数）。
* **主按钮：** 底部放置一个显眼的 **[导出视频]** 或 **[保存修改]** 按钮。





---

### 3. 具体修改建议总结（针对你现在的痛点）

#### **针对“编辑和导出分开不妥”的修改：**

* **原来的逻辑：** 步骤2（输入参数） -> 步骤3（再次确认参数并点击导出）。
* **新的逻辑：** **右侧属性面板**。
* 用户在左侧预览视频，觉得不需要裁剪，直接看右侧面板。
* 右侧面板默认显示“1080p / 30fps / MP4”。
* 用户如果想改，直接在右侧改，改完点击“导出”。
* **去重：** 既然“编辑”就是设参数，那就不要叫“编辑”了，直接叫“导出设置”或者“文件属性”。



#### **针对“界面奇怪”的视觉微调：**

* **去掉 FFmpeg 报错文本：** 把它做成一个右上角的 `Toast` 弹窗（几秒后消失）或者放在“设置”菜单里，不要常驻在界面上吓人。
* **弱化 Project ID：** 在列表页它是唯一的 Key，但在工作台，把它变成顶部的标题栏（如 `Project_20231027`），允许用户点击重命名，不要做成一个填空题。

### 4. 你的新界面线框图构想 (Text Wireframe)

**场景：录制完成后的样子**

```text
+---------------------------------------------------------------+
|  < 返回列表      项目名称: Demo_Video_01        [保存] [导出] |  <-- 顶部栏
+---------------------------------------------------------------+
|                               |                               |
|                               |   导出设置 (原编辑+导出)      |
|                               |                               |
|      [ 视频预览播放器 ]       |   分辨率: [ 1080P v ]         |
|                               |   帧率:   [ 60 FPS v ]        |
|      (在此处拖拽裁剪)         |   码率:   [ 高画质 v ]        |
|                               |                               |
|   [> 播放]  [<----|---->]     |   输出路径: D:/Videos/        |
|             时间轴滑块        |                               |
|                               |   [ 重新录制 ]                |
|                               |   [ 立即导出 ] (主按钮)       |
|                               |                               |
+---------------------------------------------------------------+

```

**这样的设计优势：**

1. **符合直觉：** 左边看结果，右边调参数。
2. **流程闭环：** 列表页管理 -> 录制 -> 预览调参 -> 导出 -> 回到列表。
3. **专业感：** 看起来像 OBS 或 剪映 的简化版，而不是一个网页表单。
